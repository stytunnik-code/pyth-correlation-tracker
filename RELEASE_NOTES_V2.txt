PYTH CORRELATION TRACKER
RELEASE NOTES V2

==================================================
1. ОБЩИЙ СТАТУС
==================================================

Текущая версия проекта переведена из состояния "визуально рабочий, но местами статистически грязный прототип" в состояние "рабочий аналитический инструмент с базовой проверкой качества данных и тестовым контуром".

Основной фокус v2:
- очистка quantitative logic;
- исправление live merge;
- устранение ложных корреляций;
- приведение entropy и lead-lag к более честной логике;
- добавление тестов;
- добавление feed diagnostics.


==================================================
2. ЧТО ИСПРАВЛЕНО В V2
==================================================

2.1. Matrix

- корреляция переведена с raw price levels на aligned returns;
- live series теперь выравниваются по timestamp;
- снижено количество ложных спурионных корреляций.


2.2. Live data pipeline

- добавлен timestamped live tick store;
- live ticks больше не склеиваются по индексу;
- history + live merge идет по реальным bucket timestamps;
- убраны fake timestamps в аналитических view.


2.3. Chart fixes

- улучшен рендер последней свечи;
- исправлены flat / doji scenarios;
- снижены ложные гэпы на HYPE;
- live marker показывается осторожнее;
- synthetic live candle logic поджата.


2.4. Correlation view

- rolling correlation теперь строится на cleaner merged flow;
- scatter / rolling view работают на returns;
- убрана грязная подрисовка live tail.


2.5. Lead-Lag

- раньше лучший лаг выбирался и оценивался на одном и том же окне;
- теперь лаг сначала ищется на train segment;
- затем проверяется на out-of-sample segment;
- ranking использует validated score, а не только peak in-sample correlation.


2.6. Entropy / NMI

- ranking теперь трактуется как Gaussian entropy of returns;
- убрана путаница с Shannon entropy в product logic;
- добавлен permutation-adjusted NMI;
- hidden connections стали менее шумными.


2.7. Math hardening

- `pearson()` фильтрует non-finite пары;
- degenerate series больше не дают fake zero correlation;
- `pctReturns()` защищён от деления на 0 и bad values;
- finite filtering добавлен в entropy / MI pipeline.


2.8. API hardening

- `/api/pyth` переведен на partial-success модель через `Promise.allSettled`;
- один упавший batch больше не валит весь response;
- добавлены API contract tests.


2.9. Feed diagnostics

- добавлен live-data sanity layer;
- проверяются stale ticks;
- проверяется timestamp skew;
- проверяется monotonic order;
- в UI добавлен badge `FEED OK / WARN / DEGRADED`;
- по клику открывается diagnostics popover с проблемными symbols.
